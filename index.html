Collatz Problem

The Collatz problem asks whether repeating two simple arithmetic operations will eventually transform every positive integer into 1. It concerns sequences of integers in which each term is obtained from the previous term as follows: if a term is even, the next term is one half of it. If a term is odd, the next term is 3 times the previous term plus 1. The conjecture is that these sequences always reach 1, no matter which positive integer is chosen to start the sequence.

Statement of the Problem

Define the function \( f(n) \) as follows:

$$ f(n) = \begin{cases} \dfrac{n}{2}, & \text{if } n \equiv 0 \pmod{2} \\ 3n + 1, & \text{if } n \equiv 1 \pmod{2} \end{cases} ,n\in\mathbb N $$

Now form a sequence by performing this operation repeatedly, beginning with any positive integer, and taking the result at each step as the input at the next.

$$ a_i = \begin{cases} n, & \text{if } i = 0 \\ f(a_{i-1}), & \text{if } i > 0 \end{cases} , n\in\mathbb N, a\in\mathbb W $$

Note: \(\mathbb W = \mathbb N\cup\{0\} = \{0,1,2,3,\ldots\}\)

Will the sequence \( a \) eventually reach 1, regardless of which \( n \) is chosen initially?

Algorithms

Naive algorithm

Let \(n_0\) be a positive integer. Assign \(n\leftarrow n_0\), then repeat the following until \(n< n_0\).
  1. if \(n\equiv0\pmod2,n\leftarrow3n+1\), else \(n\leftarrow\dfrac n2\)

Naive shortcut algorithm

Notice how for \(n\in\mathbb N\), \(3n+1\) is always an even number and will thus always divide itself by 2 in the next iteration. Therefore, we can simply divide by 2 immediately to reach the result faster.

Let \(n_0\) be a positive integer. Assign \(n\leftarrow n_0\), then repeat the following until \(n< n_0\).

  1. if \(n\equiv0\pmod2,n\leftarrow\dfrac{3n+1}2\), else \(n\leftarrow\dfrac n2\)

New algorithm used in Barina D (2021) Convergence verification of the Collatz Problem. J Super Comput 77(3):2681--2688.

Let \(n_0\) be a positive integer. Assign \(n\leftarrow n_0\), then repeat the following until \(n < n_0\).

  1. \(n\leftarrow n+1\)
  2. \(\alpha\leftarrow ctz(n)\)
  3. \(n\leftarrow\dfrac{3^\alpha n}{2^\alpha}\)
  4. \(n\leftarrow n-1\)
  5. \(\beta\leftarrow ctz(n)\)
  6. \(n\leftarrow\dfrac{n}{2^\beta}\)

The \(\text{ctz}(n)\) operation counts the number of trailing zero bits following the least significant nonzero bit in its binary representation.
